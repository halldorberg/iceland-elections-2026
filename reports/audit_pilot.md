## Patterns

1. **"Áhersla / áhugasvið" gloss is the single most common unsourced pattern.** 6 of 8 bios assert campaign priorities or "interest areas" not in any cited source. Agents seem to invent plausible policy bundles based on job title or one op-ed topic.

2. **Educational specifics (degree, university, thesis title) are often fabricated.** Sandra Hlín's quoted thesis title and Hans Alexander's "BA heimspeki + MA kynjafræði" have zero source support. Finnur's Utrecht degree relies entirely on LinkedIn (unreachable).

3. **Date/era specifics drift.** Páll Edwald's "frá sumrinu 2025" directly contradicts source (which says late-summer 2024). Guðný María's "Isavia" is anachronistic for the 2008 source. These are subtle factual errors that read fluently.

4. **Job-title elaborations get added.** Andrea Edda "námsráðgjafi" is invented (source: "stúdentaráðsliði"). Þórdís Jóna gets "markþjálfi" tacked on. Hans Alexander gets "Pírata frá 2013" tacked on.

5. **Workplace specifics get added.** Þórdís Jóna's "að Hlaðgerðarkoti" is unsourced. Finnur's "Sambandi íslenskra sveitarfélaga" employer cannot be verified from non-LinkedIn sources.

6. **List-position seat number is reliably sourced** when a party-list URL is included. The seat number is the single most consistently verifiable claim across all 8 bios.

## Time per bio

Roughly **5–8 minutes per candidate** in this pilot, dominated by:
- Source fetching (parallelisable)
- Grepping HTML for Icelandic strings (slowest step — needs careful encoding handling)
- Composing the per-claim writeup

For 570 candidates with the same workflow:
- Best case (parallelised fetches, scripted decomposition): ~20–30 minutes per 10-candidate batch → ~20–30 hours of agent time
- Realistic (with retry/encoding fixups): **40–60 hours of agent time**
- Cost estimate at current Opus pricing: meaningful — likely worth chunking and using Sonnet for the bulk pass with Opus for spot-check.

**Suggested optimisation:** pre-fetch all sources via a Python script (uses Edge UA, caches to disk), then run the agent only on locally-cached HTML. Cuts per-bio time by ~50%.
